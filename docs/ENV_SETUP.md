# YGB Environment Setup — Multi-Device Guide

## Quick Start (New Laptop/Device)

```powershell
# 1. Clone the repo
git clone <your-repo-url> C:\Users\YourName\YGB
cd C:\Users\YourName\YGB

# 2. Bootstrap environment (auto-generates secrets)
.\scripts\bootstrap-env.ps1 -DeviceId laptop_b

# 3. Edit .env — add GitHub OAuth credentials
notepad .env
# Fill in GITHUB_CLIENT_ID and GITHUB_CLIENT_SECRET

# 4. Start everything
.\start_full_stack.ps1
```

## Why One Laptop Works and Another Fails

**Root cause:** The `.env` file containing `GITHUB_CLIENT_ID` and other secrets is in `.gitignore` (correctly — secrets should never be committed). When you clone the repo on a new laptop, the `.env` file doesn't exist.

**Solution:** We commit `.env.example` (no secrets, only structure) and provide `bootstrap-env.ps1` to generate `.env` with auto-generated secrets on each device.

## Required Variables

| Variable | Description | How to get |
|----------|-------------|------------|
| `JWT_SECRET` | API JWT signing key | Auto-generated by bootstrap |
| `YGB_HMAC_SECRET` | HMAC signing key | Auto-generated by bootstrap |
| `YGB_VIDEO_JWT_SECRET` | Video streaming JWT | Auto-generated by bootstrap |
| `GITHUB_CLIENT_ID` | GitHub OAuth app ID | [github.com/settings/developers](https://github.com/settings/developers) |
| `GITHUB_CLIENT_SECRET` | GitHub OAuth secret | Same GitHub page |

## GitHub OAuth Setup

1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
2. Create/edit your OAuth App
3. Set **Authorization callback URL** to: `http://localhost:8000/auth/github/callback`
4. Copy Client ID and Client Secret to your `.env`

> **Important:** All devices sharing the same GitHub OAuth App must use the same `GITHUB_CLIENT_ID` and `GITHUB_CLIENT_SECRET`. Only the secrets (JWT, HMAC) should differ per device.

### Multi-Device OAuth Callback

For LAN access (other devices connecting to this laptop's server):

```powershell
.\start_full_stack.ps1 -BindAllInterfaces
```

The callback URL will auto-detect your network IP. Make sure to add it in your GitHub OAuth App:
- `http://localhost:8000/auth/github/callback`
- `http://192.168.x.x:8000/auth/github/callback`

## Sync Engine

The sync engine starts automatically with the full stack. Configure peer nodes in `.env`:

```ini
YGB_PEER_NODES=laptop_b:100.64.0.2:8000,laptop_c:100.64.0.3:8000
```

## Validation

Run before starting the server to check all vars are set:

```powershell
.\scripts\validate-env.ps1
```

## File Security

| File | Tracked in Git? | Contents |
|------|----------------|----------|
| `.env.example` | ✅ Yes | Template, no secrets |
| `.env` | ❌ No (.gitignore) | Real secrets |
| `secure_data/` | ❌ No (.gitignore) | Session data, keys |

## Troubleshooting

| Error | Fix |
|-------|-----|
| `GitHub OAuth not configured` | Set `GITHUB_CLIENT_ID` in `.env` |
| `JWT_SECRET is not set` | Run `.\scripts\bootstrap-env.ps1` |
| `CORS error on login` | Ensure `FRONTEND_URL` matches your browser URL |
| `Callback URL mismatch` | Add your URL to GitHub OAuth App settings |
| `Access denied from other device` | Use `.\start_full_stack.ps1 -BindAllInterfaces` |
